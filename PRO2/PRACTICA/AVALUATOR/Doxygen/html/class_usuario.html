Descripción detallada

Representa un Usuario de la plataforma.

El usuario se representa con un identificador. Algunas de las características de cada usuario son, por ejemplo, cuántos envíos en total ha realizado, cuántos problemas ha resuelto satisfactoriamente, cuántos problemas ha intentado resolver, y el curso donde está inscrito o no.

◆ intentos_problema()

void Usuario::intentos_problema ( string &  p)

Se agrega a un map (string - int) el problema con identificador p si no estaba ya previamente y se contabiliza cuantos envios ha hecho el usuario implicto a ese problema p.

Precondición
Existe el problema con identificador p.
Postcondición
Se actualiza el map anterior (string - int) con los identificadores de problemas que han sido enviados previamente al menos una vez por el usuario implicito (parametro clave del map) mas el identificadorp si no habia sido enviado antes. Tambien se actualiza el parametro asociado del map, se suma uno al contador de envios de ese problema.

◆ agregar_problema_resuelto()

void Usuario::agregar_problema_resuelto ( string &  p)

Se agrega el problema con identificador p a los problemas resueltos del usuario implícito mediante un map (string - Problema). Como se utiliza un map, estan ordenados por identificador.

Precondición
Existe el problema con identificador p.
Postcondición
El resultado es el map anterior de problemas resueltos del usuario implicto mas el problema con identificador p.

Se actualizan los parametros del usuario implicito dependiento del resultado del envio r (si es envio con exito r = 1, en caso contrario r = 0 ) de un nuevo envío al problema p del usuario implícito.

Precondición
El usuario implícito está registrado e inscrito en un curso donde aparece el problema con identificador p y que este pertenece al conjunto de problemas a los que el parámetro implícito puede enviar una solución por cumplir los prerrequisitos.
Postcondición
Se actualiza las estadísticas del parámetro implícito del problema con identificador p. Si r = 1 también tambien se actualizadan los problemas enviables y resueltos del usuario implícito. En el caso de que se haya completado el curso al que pertenece p el usuario implícito deja de estar inscrito en él.

Se escibe para el usuario implicito, su identificador, cuántos envíos en total ha realizado, cuántos problemas ha resuelto satisfactoriamente, cuántos problemas ha intentado resolver (al menos ha hecho un envío), y el identificador del curso en el que está inscrito o un 0 si no está inscrito en ninguno.
